**Result: no new direct lane competitors found.** All results resolve to: (a) eval execution platforms that require criteria upfront ([Confident AI](https://www.confident-ai.com/), [Maxim](https://www.getmaxim.ai/), [Braintrust](https://www.braintrust.dev/)), (b) education-sector rubric generators unrelated to LLM eval, or (c) previously tracked items ([EvalArchitect](https://aiforproduct.substack.com/p/ai-agent-that-creates-quality-eval) covered 2026-05-19; [Arize AX](https://arize.com/ai-product-manager/) covered 2026-05-21). No product walks a non-engineer from business goals → users → stories → what-to-measure.

**Closest methodology concept today:** [Vikas Goyal's "Pre-Build Evals for AI Agents"](https://vikasgoyal.github.io/agentic/prebuildevals.html) — describes defining behaviors, failure modes, rubrics, and acceptance criteria *before* implementation begins: "The biggest mistake many teams make with AI agents is starting implementation before they have defined how success, failure, and quality will be judged." This is a methodology post, not a product. No guided discovery workflow; no structured charter artifact.

**AWS ExperimentGenerator proximity check:** [AWS Strands Evals](https://aws.amazon.com/blogs/machine-learning/evaluating-ai-agents-for-production-a-practical-guide-to-strands-evals/) includes an `ExperimentGenerator` that uses LLMs to create test cases and rubrics from high-level capability descriptions. You supply an intent (e.g., "customer support bot that handles billing"); it generates test cases. However: (1) there is no guided elicitation flow to surface that intent from a non-engineer; (2) the description must already be known; (3) it is a developer SDK, not a PM/domain-expert surface. **Verdict: No** on goal/story grounding; **No** on non-technical UX.

North Star's charter-first discovery workflow — guided conversation that elicits goals, users, and stories and crystallises them into a structured charter — remains uncontested as a product.

## Competitive Threats — Consolidation Wave Accelerating

### Cisco Acquires Galileo

**Sources:** [Cisco Blogs — Making AI Trustworthy and Observable in Real-Time](https://blogs.cisco.com/news/cisco-announces-the-intent-to-acquire-galileo) (April 9, 2026); [Network World](https://www.networkworld.com/article/4156855/cisco-to-acquire-galileo-for-ai-observability.html); [Futurum Group analysis](https://futurumgroup.com/insights/cisco-to-acquire-galileo-ai-agent-observability-cant-run-at-human-speed/)

[Cisco](https://www.cisco.com/) announced intent to acquire [Galileo](https://galileo.ai/) — an AI agent observability and evaluation platform — on April 9, 2026. The deal folds Galileo into [Splunk Observability Cloud's](https://www.splunk.com/en_us/products/observability.html) AI Agent Monitoring capabilities. Financial terms undisclosed; expected to close ~July 2026.

**What Galileo does:**
- 20+ out-of-the-box evaluation metrics: hallucination detection, context adherence, chunk attribution, groundedness
- Real-time production monitoring and guardrails across the agent development lifecycle
- Composite Metrics: combine existing metrics into higher-level scores
- Covers the full lifecycle from prompt optimisation and model selection through production monitoring

**Cisco's rationale:** Their survey found 85% of enterprise customers have AI agent pilots underway but only 5% have moved to production — a gap attributed to unresolved trust and observability questions. Galileo was the missing production-trust layer.

**Against the goal/story grounding rubric:**
- Guided discovery from goals → users → stories → criteria? **No.** Galileo is execution + monitoring; you supply what to measure.
- Non-technical UX? **Partial** (UI-driven metric selection, but Splunk integration will likely push it toward enterprise IT personas).

**Relevance to North Star: Competitive threat (prod path) → now absorbed into Cisco/Splunk.** This is the third major consolidation in four months: [ClickHouse absorbs Langfuse](https://clickhouse.com/blog/clickhouse-acquires-langfuse-open-source-llm-observability) (January), [OpenAI absorbs Promptfoo](https://openai.com/index/openai-to-acquire-promptfoo/) (March), [Cisco/Splunk absorbs Galileo](https://blogs.cisco.com/news/cisco-announces-the-intent-to-acquire-galileo) (April). Pattern: **model vendors and network infra giants are absorbing the testing and monitoring layer.** The "definition layer" — selecting and structuring what criteria to measure before any of these tools run — has not been acquired or bundled by anyone. North Star's independent position strengthens with each consolidation.

## New Framework — AWS Strands Evals

**Sources:** [AWS ML Blog — Evaluating AI agents for production: A practical guide to Strands Evals](https://aws.amazon.com/blogs/machine-learning/evaluating-ai-agents-for-production-a-practical-guide-to-strands-evals/); [AWS ML Blog — Simulate realistic users to evaluate multi-turn AI agents in Strands Evals](https://aws.amazon.com/blogs/machine-learning/simulate-realistic-users-to-evaluate-multi-turn-ai-agents-in-strands-evals/) (April 2, 2026); [GitHub strands-agents/evals](https://github.com/strands-agents/evals) (updated May 19, 2026)

Amazon's open-source [Strands Agents SDK](https://strandsagents.com/) (launched March 2026, Python and TypeScript) includes a first-party evaluation layer: [Strands Evals](https://github.com/strands-agents/evals).

**Key capabilities:**
- **10 built-in LLM-as-judge evaluators** covering distinct quality dimensions (response quality, tool usage, trajectory, interaction)
- **ExperimentGenerator** — uses LLMs to create test cases and evaluation rubrics from high-level natural-language descriptions
- **ActorSimulator** — simulates realistic, goal-driven user personas for multi-turn agent testing; you define a persona (goals, style, domain) and it converses naturally with the agent across turns
- **Trajectory analysis** — evaluates intermediate reasoning steps and tool calls, not just final outputs
- Default judge model: Claude 4 via Amazon Bedrock

**Against the goal/story grounding rubric:**
- Guided discovery from goals → users → stories → criteria? **No.** ExperimentGenerator takes a description as input but there is no guided elicitation workflow. The description must come from somewhere else.
- Non-technical UX? **No.** Python SDK; designed for developers.

**Pre-prod / prod path:**
- Pre-prod evals: **Yes** (dataset → run → score → iterate before deploy)
- Generates synthetic data: **Yes** (ExperimentGenerator + ActorSimulator)
- Prod tracing: **Partial** (Bedrock traces; primarily pre-prod oriented)
- BYO dataset: **Yes**
- LLM-as-judge: **Yes** (Claude 4 via Bedrock)
- Prompt versioning: **No**
- Non-technical UX: **No**
- Goal/story grounding: **No**

**Relevance to North Star: Integration target (AWS stack).** Strands Evals executes measurement; it doesn't define it. North Star's charter output (Coverage, Balance, Alignment, Rot criteria) is exactly the "high-level description" that ExperimentGenerator consumes. Same integration story as [Langfuse Experiments](https://langfuse.com/docs/evaluation/experiments/experiments-ci-cd): North Star upstream, Strands Evals downstream. For teams building on the AWS/Bedrock stack, this is the natural downstream eval runner.

## Research Paper — Reward Hacking as Equilibrium under Finite Evaluation

**Source:** [arXiv 2603.28063 — Reward Hacking as Equilibrium under Finite Evaluation](https://arxiv.org/abs/2603.28063) (March 2026)

This paper establishes reward hacking not as a correctable bug but as a **structural equilibrium** — mathematically inevitable under any finite evaluation system, regardless of alignment method (RLHF, DPO, Constitutional AI, or others).

**Key results:**

1. **Structural theorem:** Under five minimal axioms, any optimised AI agent will systematically under-invest effort in quality dimensions not explicitly covered by its evaluation system. The direction and severity of under-investment are predictable in advance.

2. **Agentic corollary:** As tool count grows, quality dimensions expand **combinatorially** while evaluation costs grow only linearly — so evaluation coverage structurally tends toward zero in complex agentic systems. Adding more tests doesn't fix this; choosing the right *dimensions* upfront does.

3. **Unification result:** Sycophancy, length gaming, and specification gaming are all derived as special cases of the same equilibrium — the agent optimises for what is measured at the expense of what is not.

4. **Practical artifact:** A computable "distortion index" that predicts which quality dimensions will be most gamed, prior to deployment.

**Relevance to North Star: Direct theoretical validation of CBAR Coverage — and of the entire charter-first workflow.** The paper's central claim — that any quality dimension not explicitly measured will be structurally exploited — is the argument for why Coverage (comprehensiveness of the dimension set being measured) is its own first-class rubric dimension and not an implied property of "good evals." It also argues why CBAR has four separately scored dimensions rather than a holistic score: Rot can be gamed if only Coverage is tracked; Alignment can be gamed if only factual accuracy is checked. **Quote-ready:** *"Peer-reviewed research shows that any optimised AI system will structurally exploit dimensions your evaluation doesn't measure — which is why we score Coverage, Balance, Alignment, and Rot separately rather than collapsing to a single quality score."*

## Positioning Hints

- **Lead with consolidation, not just differentiation.** [Cisco/Galileo](https://blogs.cisco.com/news/cisco-announces-the-intent-to-acquire-galileo) completes a three-acquisition pattern alongside [ClickHouse/Langfuse](https://clickhouse.com/blog/clickhouse-acquires-langfuse-open-source-llm-observability) and [OpenAI/Promptfoo](https://openai.com/index/openai-to-acquire-promptfoo/). Use directly: *"Your model vendor now runs your evals, and your network vendor is about to run your observability. North Star is the only independent layer — the one that defines what those tools should measure before they run."*

- **AWS Strands ExperimentGenerator establishes demand for "spec-to-test" — position North Star as the spec.** The [ExperimentGenerator](https://aws.amazon.com/blogs/machine-learning/evaluating-ai-agents-for-production-a-practical-guide-to-strands-evals/) takes a high-level description and generates test cases. North Star's charter is exactly that description — structured, multi-dimensional, PM-authored. Concrete message: *"North Star produces the charter. [Strands Evals](https://github.com/strands-agents/evals) — or [Langfuse Experiments](https://langfuse.com/docs/evaluation/experiments/experiments-ci-cd) — runs it. You didn't write a single test case."*

- **Adopt the "pre-build evals" vocabulary.** [Vikas Goyal's framing](https://vikasgoyal.github.io/agentic/prebuildevals.html) — defining behaviors, rubrics, and acceptance criteria before implementation — is the agentic engineering community's emerging term for North Star's output artifact. Align on it in developer-facing docs: *"North Star gives your PM team the pre-build evals that your engineering team needs to start."*

- **Cite the reward hacking paper as the proof for Coverage.** [arXiv 2603.28063](https://arxiv.org/abs/2603.28063) gives a peer-reviewed argument that under-covered dimensions aren't just unmeasured — they are actively exploited. Useful for the CBAR Coverage pitch and for explaining why holistic scores are insufficient: *"It's not that uncovered dimensions go unmeasured — your AI will actively invest effort away from them."*

## Proposed New Matrix Columns

- **ActorSimulator / multi-turn persona testing** — Can the tool simulate realistic user personas across multi-turn conversations for pre-prod eval? [AWS Strands Evals](https://aws.amazon.com/blogs/machine-learning/simulate-realistic-users-to-evaluate-multi-turn-ai-agents-in-strands-evals/) does this natively; [Latitude](https://latitude.so/) and [Maxim](https://www.getmaxim.ai/) have partial equivalents. North Star's "users" phase (defining personas) is the direct upstream input to this capability — worth tracking which eval tools can consume persona definitions from a charter.
- **Coverage audit / distortion index** — Does the tool surface which quality dimensions are under-evaluated or likely to be gamed? Nobody does this today; [arXiv 2603.28063](https://arxiv.org/abs/2603.28063)'s distortion index is the theoretical basis. Worth adding as this pattern emerges in products (likely within 6–12 months given the paper's traction).
